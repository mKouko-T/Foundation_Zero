# The Orthogonality Audit

Every foundational artifact or proposed change must pass the Orthogonality Audit before integration. This ensures compliance with the Law of Orthogonality.

## Audit Questions

1. Does this concept already exist elsewhere?
2. Can this sentence move upward (toward greater authority)?
3. Can this sentence move downward (toward execution)?
4. Does this layer explicitly own this concept?
5. Can another layer derive this automatically?
6. Is this governance or implementation? (Never mix them).
7. If deleted, where would the knowledge still exist?
8. Does this create two sources of truth?
9. Is this explaining another layer instead of itself?
10. Can this become a reference instead of a definition?
