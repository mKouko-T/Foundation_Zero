# Layer C: Foundational Principles

Principles interpret the Constitution. They guide how we apply timeless truths to the act of building, reasoning, and engineering.

## Principle 000: The Separation of State
Separate the timeless (principles, standards, methodology) from the temporary (chats, tools, versions). Timeless knowledge shall never be polluted by temporary information.

## Principle 001: The Locus of Complexity
Complexity belongs inside the system, not inside the user. If using the system feels complicated, the architecture has failed. Good infrastructure makes difficult work feel straightforward.

## Principle 002: The Evolution of Methodology
The methodology itself is always subject to improvement, but only through evidence, clearer reasoning, or better stewardship—not through novelty alone. We evolve deliberately to avoid the extremes of never changing and constantly changing.

## Principle 003: The Priority of Clarity
Every page we write must reduce future confusion. If a document creates more questions than it resolves, it isn't ready. If a system only works because we remember hidden context, it isn't complete. Complexity impresses; clarity endures.

## Principle 004: Graceful Degradation
Good systems fail gracefully. The Foundation Method methodology must survive the loss of any specific tool. If Antigravity, ChatGPT, Notion, Obsidian, or the internet disappears, the whole system must still function. The methodology lives in the reasoning, not the software.
