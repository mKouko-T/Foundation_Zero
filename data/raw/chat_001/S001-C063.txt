# Repository Constitution Test
Before any governance modification is accepted, it must pass this 7-point test:
1. [ ] Does this reduce ambiguity?
2. [ ] Does this improve traceability?
3. [ ] Does this reduce cognitive load?
4. [ ] Does this remain reusable across projects?
5. [ ] Does this preserve orthogonality?
6. [ ] Could this have been solved by improving an existing artifact instead of creating a new one?
7. [ ] Will this still make sense after 100 sessions?
**Rule:** Only if all seven pass should the governance change be accepted. This prevents Governance itself from becoming bloated over time.

# QG-001: Decision Quality Gate
Before a decision is accepted, ensure:
- [ ] Is the statement clear and unambiguous?
- [ ] Are alternatives considered and rejected for logical reasons?
- [ ] Is there verifiable evidence backing this decision?
- [ ] Are reversal conditions explicitly defined?
- [ ] Are impacted artifacts and dependencies identified?
# QG-002: Evidence Quality Gate
Before evidence is accepted, ensure:
- [ ] Is the reality observed directly?
- [ ] Is the measurement quantifiable and objective?
- [ ] Are inferences separated from the base measurement?
- [ ] Does this evidence address the core operational bottleneck?
# QG-003: Repository Synchronization Gate
Before a synchronization packet is considered canonical, ensure:
- [ ] Are there zero duplicate files?
- [ ] Are there zero duplicated headings with semantic overlap?
- [ ] Is there exactly one version of each artifact present?
- [ ] Is the packet split strictly into "Repository Snapshot" and "Synchronization Instructions"?
# QG-004: Operational Brief Quality Gate
Before an Operational Intelligence Brief is published, ensure:
- [ ] Is every recommendation traceable to evidence?
- [ ] Is every inference supported by observations?
- [ ] Are hypotheses clearly labeled?
- [ ] Does the Brief require an executive decision?
- [ ] Would the organization behave differently tomorrow if this Brief were read?
- [ ] Can success or failure be measured objectively?
- [ ] Would the Brief still be understandable six months from now without conversational context?
# QG-005: Canon Addition Gate
Before any process, decision, or artifact is elevated to canonical status, ensure:
- [ ] Has it been validated by at least three successful operational interventions?
- [ ] Did it produce better executive decisions?
- [ ] Is it reusable across the organization (orthogonal to any specific project)?
