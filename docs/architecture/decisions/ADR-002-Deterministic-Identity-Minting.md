# ADR-002: Deterministic Identity Minting

**Status:** Accepted
**Date:** 2026-07-11

## Context
If UUIDs are generated randomly during automated pipeline runs, identical inputs on different runs will create separate logical entities, forcing an impossible merge problem.

## Decision
All `FZ-` IDs must be generated by an `IdentityProvider` interface. When determinism is required, identity is a canonical JSON string (RFC-8785) hash of the objective inputs. Identical inputs will mathematically guarantee identical IDs.

## Consequences
- Idempotency is structurally guaranteed.
- Replaying pipelines will never duplicate records.
