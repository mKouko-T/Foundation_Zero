# Foundation Zero: Constitutional Architecture

*Foundation Zero is a Computational Constitution. It defines the immutable mathematical laws under which new objects, relationships, and reasoning emerge safely, without dictating infinite domain-specific ontologies.*

## The Four-Layer Architecture
To prevent abstractions from bleeding into implementations, the system strictly isolates its conceptual foundations into four hierarchical levels.

### Level 0: The Constitutional Layer (Immutable Laws)
The absolute, unchanging mathematical constraints of the universe.
- **Scope**: Axioms of identity, monotonic history, provenance continuity, uncertainty propagation, contextual relativity, and explicitness.
- **Example**: "No Entity may exist without an unbroken Causal chain."

### Level 1: The Meta-Model (Computational Substrate)
The generic mathematical machinery that powers the system.
- **Scope**: The 3 Irreducible Primitives (Identity, Relation, State), Identity Algebra, the Graph structures, and Orthogonal Facets.
- **Example**: Defining how an Identity can Fork into two Identities.

### Level 2: Domain Ontologies (Semantic Reality)
The specific applications of the Meta-Model to solve real-world problems. **Identity does not replace Ontology.** Identity = *WHO*. Ontology = *WHAT*. State = *CURRENT CONDITION*.
- **Scope**: The Knowledge Ontology (Assertion, Observation, Evidence), Governance Ontology (Decision, Protocol), Software Ontology (Capability, Code).
- **Example**: Declaring that Identity `0x123` is classified as an `Observation` in the Epistemic Facet.

### Level 3: Operational Implementations
The actual tools, databases, and workflows executing the system.
- **Scope**: Migration engines, graph databases, Git repositories, JSON payloads, and user applications.
- **Example**: `fz certify` executing a cryptographic hash of the JSON state.

## Constitutional Governance (The Amendment Process)
To protect Foundation Zero against infinite constitutional bloat, the following strict governance protocols apply:
1. **Implementation Imperative**: No further conceptual milestones may be proposed without an accompanying implementation milestone. A constitutional amendment must justify itself by demonstrating a concrete implementation failure that cannot be solved within the existing constitution.
2. **Level 0 Additions**: A Level 0 law is virtually impossible to add. It requires absolute mathematical proof of its necessity.
3. **Level 1 Primitives**: A new Level 1 primitive requires proof that it cannot be expressed as a composition of existing primitives.
4. **Default to Level 2/3**: Everything else strictly belongs in Level 2 (Domain Ontologies) or Level 3 (Implementations).
