# Repository Smells

**Definition:** A smell is a heuristic. It is a weak signal that something *might* be wrong, but it is not necessarily a direct violation of the rules. Smells indicate decay, confusion, or friction.

Unlike Anti-Patterns (which are known, proven failures), smells are subjective warnings that require human or agentic investigation.

## Common Smells

1. **File nobody can explain.**
   - A document or script that exists, passes tests, but no current engineer or playbook understands its origin or purpose.
2. **Playbook nobody executes.**
   - A process defined in `docs/playbooks/` that hasn't been run or tested in months.
3. **Skill nobody loads.**
   - An AI capability or tool that is registered but never actually invoked during execution.
4. **ADR nobody references.**
   - An architectural decision record that was written but is never cited as the Authority for any subsequent code or documentation.
5. **Document whose purpose changed.**
   - A file named `setup_guide.md` that is now exclusively used to store database credentials.
6. **Two documents competing to be authoritative.**
   - `architecture.md` says one thing, while `design.md` says another. Neither explicitly supersedes the other.
7. **Generated artifact edited manually.**
   - A JSON projection or autogenerated report containing manual, human-typed fixes that will be destroyed on the next compilation.
8. **CI warning ignored for >30 days.**
   - A non-fatal warning generated by the Verification Suite that everyone has become numb to.
