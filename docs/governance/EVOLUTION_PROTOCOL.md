# Evolution Protocol

Architecture must never evolve through speculation or conversation. It must evolve exclusively through physical experimentation.

## The Mandatory Sequence

1. **Idea:** A hypothesis for an improvement.
2. **Prototype:** A temporary, disposable execution outside the kernel.
3. **Real Implementation:** An attempt to use the prototype to solve a real business decision.
4. **Failure Analysis:** Measurement of where the prototype succeeds or fails.
5. **Architecture:** (Only if necessary) Formalizing the validated pattern into the repository.

Architecture is always the **last** step, never the first.
