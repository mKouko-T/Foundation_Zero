# Foundation Zero: Recovery Registry

During the Repository Reconstitution (Milestone 3), the system operates strictly under the **Recovery Rule**: *No new concept is added simply because it seems better.* 

Any insight discovered during implementation or reconstruction must be formally classified into one of three categories:
- **Type A (Forgotten Knowledge)**: Existed in prior architecture iterations but was lost. (Status: Accepted)
- **Type B (Clarification)**: Exists implicitly in the Constitution but requires formalization. (Status: Accepted)
- **Type C (New Idea)**: A novel proposal that expands the Constitution. (Status: Deferred until post-Milestone 3).
